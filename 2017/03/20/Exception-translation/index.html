Exception translation

当在DAO中使用Hibernate，JPA，JDO时，要处理一些持久层技术的原生异常类型，比如HibernateException,PersistenceException,JDOException，取决于使用哪种持久层技术。这些运行时的异常不用声明或者catch。Spring提供了一种透明的异常类型转译的方法，通过@Repository注解。

预处理器ersistenceExceptionTranslationPostProcessor会自动的扫描到所有的异常转译类，这些类都实现了PersistenceExceptionTranslator接口，并且会根据@Repository注解的DAO类的抛出异常选择恰当的异常处理方式。

当然我们可以不用选择定制直接使用原本的exception translator，但这对理解Spring的ORM技术是有帮助的。
